Endpoint

```
POST /make-server-4010be06/signout
```

Signs out the current user and invalidates their access token.

---

Headers

<ParamField header="Authorization" type="string" required>
  Bearer token to invalidate
  
  ```
  Authorization: Bearer YOUR_ACCESS_TOKEN
  ```
</ParamField>

---

Response

<ResponseField name="success" type="boolean">
  `true` if sign out was successful
</ResponseField>

---

Example Request

```typescript
const response = await fetch('https://YOUR_PROJECT.supabase.co/functions/v1/make-server-4010be06/signout', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
})

const data = await response.json()

if (data.success) {
  // Clear local storage
  localStorage.removeItem('access_token')
  
  // Redirect to landing page
  window.location.href = '/'
}
```

Complete Sign Out Flow

```typescript
import { shutdownIntercom } from '../utils/intercom'

async function handleSignOut() {
  try {
    // 1. Call sign out endpoint
    const response = await fetch('/signout', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${accessToken}`
      }
    })
    
    if (!response.ok) {
      throw new Error('Sign out failed')
    }
    
    // 2. Clear local state
    localStorage.removeItem('access_token')
    localStorage.removeItem('user_profile')
    
    // 3. Shut down Intercom
    shutdownIntercom()
    
    // 4. Clear any cached data
    sessionStorage.clear()
    
    // 5. Redirect to landing page
    window.location.href = '/'
    
  } catch (error) {
    console.error('Sign out error:', error)
    // Still clear local data even if API call fails
    localStorage.clear()
    window.location.href = '/'
  }
}
```

---

Notes

<Warning>
Always call shutdown functions when signing out:
Clear `localStorage` and `sessionStorage`
Shut down Intercom: `shutdownIntercom()`
Clear Sentry user: `Sentry.setUser(null)`
Reset analytics: `analytics.reset()`
</Warning>

<Tip>
Even if the API call fails, clear local storage to ensure user is logged out on the frontend.
</Tip>

<Info>
After sign out, the access token is invalidated server-side and cannot be reused.
</Info>
